## Preference tuning
+ 为什么要Preference tuning?
例子:假如你的模型经过下面训练后
$\text{Input: Suggest a new activity I could do with my teddy bear. }\rightarrow \text{LLM}_{SFT}$ 
这个时候LLM由于训练的数据 或许回答:$\text{I'd suggest you do not spend much time with your teddy bear at all}$❌
这显然不是我们想要的 或许我们想要的是如下的回答:✔
$\text{Of course! Teddy bears not only make awesome companions for a delightful sleep,}$
$\text{but can also be great buddies for fun activities. How about you both watch a movie together?}$
这两个回答我们称作 **preference pairs**

事实上:如果想要获得后面的answer 我们也可以训练 但是high quality dataset的代价是昂贵的
而**Comparing (e.g. A better than B) is easier than Generating (e.g. generate A from scratch)**
只要让模型生成多个回答看哪个更好就行了

而且**对SFT来说 他的Dataset很容易mess up** :90%都是coding数据 那么你的诗歌能力大幅下降 你需要精心设计高quality的dataset
但是**Not scalable: data quality is very important and hard to get** 你没有那么容易设计数据 设计一百条精心的数据集比你在网站爬一万条数据难得多
实际上往往**通过模型的misbehavior** 你也能取判断data的quality

RL overwiew:
Reward modeling + Reinforcement Learning
**Reward modeling:**
这个模型就是根据我们哪个answer更好哪个answer更坏(Iinput) 来为我们进行打分(output)
我们先要引入一个Bradley-Terry formulation
Probability that better than is defined as: 
$p(y_i > y_j) = \frac{e^{r_i}}{e^{r_i} + e^{r_j}} = \sigma(r_i - r_j)$
就是sigmod的函数里面套两个奖励的差值 正就大于0.5 反之小于0.5
![[image27.png]]
我们的reward model 就是输入我们的preference pairs 然后输出每个answer的分数
如何训练呢?
我们很容易推出
我们想要$\max \prod_{i=1}^{n} P(\hat{y}_w > \hat{y}_l)$ wining example大于lost example的概率乘积最大
带入式子和取对数 就等价于 $\log \left( \prod_{i=1}^{n} \sigma \left( r(x, \hat{y}_w) - r(x, \hat{y}_l) \right) \right)$
那么取负数 我们目标就等价于 $\Rightarrow \mathcal{L} = - \sum_{i=1}^{n} \log \left( \sigma \left( r(x, \hat{y}_w) - r(x, \hat{y}_l) \right) \right)$
这就是我们的**Loss Function:**
$$\mathcal{L}_{\theta} = - \mathbb E[ \log \left( \sigma \left( r(x, \hat{y}_w) - r(x, \hat{y}_l) \right) \right)]$$

为什么我们不想要他离base model太远?
一方面是我们的Reward model是imperfect的 agent可能会钻空子去刷分 导致我们的并不能达到想要的结果 称作**Reward hacking**  另外一方面是要保证训练的Stability
因此我们就提出了**PPO ( Proximal Policy Optimization )**
其中Proximal就告诉我们Don't deviate too much from base model
因此我们可以这样去设计函数
最大化reward + regularization term 这个正则项就用KL散度来衡量我们训练前后的policy的距离
$$\mathcal{L}(\theta) = - \left[ r(x, \hat{y}) - \lambda \mathbf{KL}(\pi_{\theta}(\hat{y}|x) || \pi_{\text{ref.}}(\hat{y}|x)) \right]$$
$\pi_{\text{ref.}}(\hat{y}|x)$ 就是我们的sft model的参数

---
但是我们在PPO里面 并不直接用reward
因为我们可以假设这样一个情景 LLM生成了50个token RM给了10分 但是可能是因为前面回答的烂 扣了10分 后面token好 加了15分 那PPO去加10分到底把功劳归给哪一个token?
这就引入了**High variance:奖励信号非常稀疏（最后才有）且模糊不清**
因此我们引入一个baseline 也就是Value function
如果我按照现在的 Policy 继续下去，我**平均**能得到多少奖励？
那么我们的**Advantage = 实际拿到的奖励 (Reward) - 期望的奖励 (Baseline)**
(严谨给一下Advantage的Def.后面强化学习数学专题在细讲:严格定义是在状态$s_t$的情况下 做出action $a_t$ 比其他动作能带来多少优势 )
$$A_{\theta}(s_t,a_t)=Q_{\theta}(s_t,a_t)-V_{\theta}(s)=r_t+\gamma\cdot V_{\theta}(s_{t+1})-V_{\theta}(s_t)$$
为啥要这么做 因为有可能你同样生成两个token A,B 奖励值一样 但是生成A导致你后面的Value function变得巨差无比 而生成B却能继续生成更好的文章
然后我们通过**GAE算法来估算我们的Advantage** 在这里不多讲解
本质就是通过算法然后用估算Value function 的神经网络估算出我们的Advantage

---
PPO Variant:现在有最常见的两种变形
- **PPO-Clip**

  本质上就是 prevent large updates，防止因为你偶然获得巨大优势，更新太大而导致局部最优。

  $$
  \mathcal{L}^{CLIP}(\theta) = \hat{\mathbb{E}}_{t} \left[ \min(r_{t}(\theta)\hat{A}_{t}, \operatorname{clip}(r_{t}(\theta), 1-\epsilon, 1+\epsilon)\hat{A}_{t}) \right]
  $$

  $$
  r_{t}(\theta) = \frac{\pi_{\theta}(a_{t}|s_{t})}{\pi_{\theta_{\text{old}}}(a_{t}|s_{t})}
  $$

  **old means Model from previous RL iteration**

  注意: 这里的 L 并不代表 loss function，准确地说我们这里想要 maximize 它。

  然后这里的 r 并不代表 reward，**而是代表 ratio**。比率 $r$ 衡量了我们的新策略 $\pi_\theta$ 在 state $s_t$ 时，采取 action $a_t$ 的概率，相对于旧策略 $\pi_{\theta_{old}}$ 而言，是**变高了还是变低了**。大于 1 就代表 $a_t$ 的概率提高了，反之则证明降低了。

  然后我们来理解这个式子。我们先考虑 A 大于 0 的情形，本质上就是得到了这样的一个函数图像。

  ![[image43.png|170]]

  我们不想让他更新太大，而是限制在 $1+\varepsilon$ 这个区间。(注意这个图像就是 min(·,·) 的图像) 超过这个区间了，那么梯度为 0 就不会更新了。

  反之 A 小于 0 的情形。

  ![[image44.png|176]]

  这个 action 很差，但是你不能直接把概率置为 0。允许策略在“小步”内自由学习，但如果某次更新试图“迈出”这个安全区域（$r > 1+\epsilon$ 或 $r < 1-\epsilon$），裁剪 (Clip) 机制就会介入，将梯度设为 0，从而**阻止这次过大的更新**。

- **PPO-KL Penalty**

前面可以看作硬手段裁剪来防止模型跑偏，这个就是看作软手段(惩罚)来约束模型。

$$
  \mathcal{L}^{KLPEN}(\theta) = \hat{\mathbb{E}}_{t} \left[ \frac{\pi_{\theta}(a_{t}|s_{t})}{\pi_{\theta_{\text{old}}}(a_{t}|s_{t})} \hat{A}_{t} - \beta \mathbf{KL}[\pi_{\theta_{\text{ref}}}(\cdot|s_{t}), \pi_{\theta}(\cdot|s_{t})] \right]
$$
前面就是最大化奖励，后面就是 LLM 不要跑偏我们 **SFT（监督微调）之后的原始模型**。
在早期 PPO 论文中，KL 约束通常是针对 $\pi_{old}$（上一轮的模型）。但在 ChatGPT / InstructGPT 这种现代 LLM 训练中，KL 几乎总是针对 **$\pi_{ref}$ (最开始的那个基座模型)**。

---
PPO很明显的弊端也显现出来
你需要拉来**四个模型**
Policy Model(Actor): 负责生成回答(也就是我们的目标训练模型)
Value Model(Critic) : 裁判 预估当前的State 未来可以得到多少分数
Base Model: 防止Actor练的太远了 用KL来约束一下两者分布
Reward Model: 打分 代替人类给最终生成的答案打分


BON - Best of N
假如我们有RL model 但是不想做RL
给定同样的prompt 在Inference阶段多做几次response 交给我们的Reward model打分 然后我们去选分数最高的回答
● Given a prompt, generate several outputs with SFT model 
● Rank output with score given by reward model 
● Take the best one

**本质上利用更多次的Inference cost来换取RL 的training cost**

## DPO
考虑到PPO的各种局限性 我们思考 为什么我们不尝试train in a supervised fashion 
这就引入了我们的DPO = Direct Preference Optimization
我们把Loss function 用一个supervised way来重写:
$$
\mathcal{L}_{\text{DPO}}(\pi_{\theta}; \pi_{\text{ref}}) = -\mathbb{E}_{(x,y_w,y_l) \sim \mathcal{D}} \left[ \log \sigma \left( \beta \log \frac{\pi_{\theta}(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_{\theta}(y_l|x)}{\pi_{\text{ref}}(y_l|x)} \right) \right]
$$
不再需要reward 直接在preference data上面就要可以做
实际上我们可以把这个看作有一个特殊reward的Bradley-Terry formulation $r_{\theta}(x, y) = \beta \frac{\pi_{\theta}(y|x)}{\pi_{\text{ref}}(y|x)}$ 
下面我们来看我们如何从PPO来得到DPO
1)我们从PPO objective开始:
$$\max_{\pi_{\theta}} \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_{\theta}(y|x)} \left[ r_{\phi}(x,y) - \beta \mathbf{D}_{\mathbf{KL}}[\pi_{\theta}(y|x) || \pi_{\text{ref}}(y|x)] \right]$$
2)现在我们先不忙跑梯度下降  而是去思考这个问题的解析解 数学上我们可以证明 上述的最优解$\pi^*$ 有一个固定的形式 
$$\pi^{\ast}(y|x) = \frac{1}{Z(x)}\pi_{\text{ref}}(y|x) \exp\left(\frac{1}{\beta}r^{\ast}(x,y)\right)$$
**$Z(x)$** 是归一化常数（Partition Function），为了保证概率之和为 1
最优的策略 $\pi^*$ 其实就是把原来的基座模型 $\pi_{\text{ref}}$ 按照奖励 $r$ 进行加权放大 奖励越高，概率越大
3)在PPO中 我们用显式的Reward Model告诉$\pi$ 怎么去做 这里我们根据上面的式子 可以发现他可以反过来把r给解出来
同时取log:$$\begin{aligned} \log \pi^*(y|x) &= \log \frac{1}{Z(x)} + \log \pi_{\text{ref}}(y|x) + \frac{1}{\beta} r^*(x,y) \\ \frac{1}{\beta} r^*(x,y) &= \log \pi^*(y|x) - \log \pi_{\text{ref}}(y|x) + \log Z(x) \end{aligned}$$
$$r^{\ast}(x,y) = \beta \log \frac{\pi^{\ast}(y|x)}{\pi_{\text{ref}}(y|x)} + \beta \log Z(x)$$
如果你给我一个policy 我就能反推出来替他通过优化什么奖励得出来的
4)那么现在我们带入我们的Bradley-Terry Model
$p(y_w > y_l | x) = \sigma(r^*(x, y_w) - r^*(x, y_l))$
$$p^{\ast}(y_w > y_l|x) = \frac{1}{1 + \exp \left( \beta \log \frac{\pi^{\ast}(y_l|x)}{\pi_{\text{ref}}(y_l|x)} - \beta \log \frac{\pi^{\ast}(y_w|x)}{\pi_{\text{ref}}(y_w|x)} \right)}$$
5)现在同样的我们做MLE 就可以做出来DPO的Loss
$$\mathcal{L}_{\text{DPO}}(\pi_{\theta}; \pi_{\text{ref}}) = -\mathbb{E}_{(x,y_w,y_l) \sim \mathcal{D}} \left[ \log \sigma \left( \beta \log \frac{\pi_{\theta}(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_{\theta}(y_l|x)}{\pi_{\text{ref}}(y_l|x)} \right) \right]$$
![[image31.png|550]]
**DPO让强化学习变得和SFT一样简单稳定 显存也显著下降**
**但是DPO也有局限性,他太依赖SFT basemodel 因为他本质上是在做对比学习(拉近正样本 推理负样本) 并不是真的的在强化探索 如果basemodel 没有生成过某种高分答案 那么DPO是很难凭空造成出来的**
